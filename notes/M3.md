   Plan: Add Login Flow to applied-ai-sandbox

   Context

   The sandbox app currently has no auth — all notes are shared in a single in-memory list. The goal is to add register/login/logout so each user sees only their own notes. The critical constraint: existing pytest tests (test_task_01, test_task_02) make unauthenticated requests and manipulate app.notes directly, and we cannot edit tests/.

   The Compatibility Trick

   Flask-Login has a LOGIN_DISABLED config flag it checks at request time inside @login_required. We add a before_request hook that sets app.config["LOGIN_DISABLED"] = True whenever app.config.get("TESTING") is True. The conftest sets TESTING=True before any requests, so on the first request the hook fires and login is bypassed for all existing tests. Anonymous current_user (is_authenticated == False) is then used as a signal to fall back to the raw app.notes list for note-scoping.

   models.py Outline

   from flask_sqlalchemy import SQLAlchemy         
   from flask_login import UserMixin
   from werkzeug.security import generate_password_hash, check_password_hash

   db = SQLAlchemy()                              

   class User(UserMixin, db.Model):                  
       id           = db.Column(db.Integer, primary_key=True)
       username     = db.Column(db.String(80), unique=True, nullable=False)
       password_hash = db.Column(db.String(200), nullable=False)
       def set_password(self, raw): self.password_hash = generate_password_hash(raw)
       def check_password(self, raw): return check_password_hash(self.password_hash, raw)
